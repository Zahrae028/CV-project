CV Builder – Stepper Form Project
Project Context

This project aims to develop a multi-step CV creation web application that allows users to input, preview, and customize their professional resume easily. The application guides users through structured steps to collect personal and professional data, and then generates a downloadable, customizable CV.

Key Features
1. Stepper Form with Validation

Personal Information: Full name, profile photo, email, phone, address, LinkedIn, GitHub, portfolio, etc.

Professional Details: Job title and personal summary.

Skills: Separate sections for technical and soft skills, with dynamic field addition.

Languages: Add languages and proficiency levels dynamically.

Hobbies & Interests: Add or remove entries dynamically.

Education: Add multiple academic records dynamically.

Work Experience: Add multiple professional experiences dynamically.

Certifications: Add certifications with name and link dynamically.

2. Progress Tracking

A visual progress bar displays the user’s advancement through the form steps.

3. Customizable CV Templates

Two distinct CV design templates to choose from.

Real-time preview and customization before printing or downloading.

4. Save Options

Optionally save created CVs in the app for future use or updates.

5. Field Validation

Built-in data validation for all input types (email, phone, URLs, etc.) to ensure data accuracy.

Technologies Used

HTML5

TailwindCSS

JavaScript (Vanilla DOM Manipulation)

User Stories and Acceptance Criteria
1. CV Creation

Users can create a CV by filling in a guided multi-step form with personal and professional details.
Acceptance criteria:

Steps are clearly defined and navigable without losing entered data.

2. Stepper Form Navigation

Acceptance criteria:

Users can move forward and backward between steps.

A progress bar visually tracks form completion.

3. Dynamic Forms

Acceptance criteria:

Users can dynamically add or remove entries for skills, languages, education, experience, and certifications.

4. Field Validation

Acceptance criteria:

Real-time validation ensures correct formats (email, phone, URLs, etc.).

Error messages appear clearly under invalid fields.

5. Progress Bar

Acceptance criteria:

Updates dynamically as steps are completed or revisited.

6. Template Selection

Acceptance criteria:

Users can preview and switch between two CV templates before download.

7. Download and Print

Acceptance criteria:

Users can download a PDF version or print their CV directly.

The PDF retains the chosen layout and formatting.

Bonus: Export the CV data as a JSON file before PDF generation.
